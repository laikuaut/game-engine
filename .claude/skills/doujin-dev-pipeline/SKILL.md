# doujin-dev-pipeline — 開発パイプラインスキル

指示されたタスクを6段階で完遂する。各ステップは個別スキルとして定義されている。

## 事前チェック

1. `git status` で未コミット変更を確認し、ある場合はユーザーに警告
2. TodoWrite で6項目を登録

## パイプライン

以下のスキルを **この順番で** 実行する。各スキル完了後に TodoWrite を更新する。

### 1. `/doujin-design <タスク説明>`
設計。影響範囲を特定し実装方針を決定する。

### 2. `/doujin-implement`
実装。設計結果に基づきコードを書く。

### 3. `/doujin-desk-check`
机上確認。実装をセルフレビューし問題があれば修正する。

### 4. `/doujin-test-create`
テスト作成。変更に対するテストを作成/更新する。テスト不要なら理由を述べてスキップ。

### 5. `/doujin-test-run`
テスト実行。全テストをパスするまで修正・リトライ（上限2回）。

### 6. `/doujin-doc-update`
ドキュメント修正。CLAUDE.md、docs/design/*.md を必要に応じて更新する。

## 完了報告

```
## パイプライン完了

### タスク
（元のタスク説明）

### 変更ファイル
- ファイル一覧

### テスト結果
- パス: X件 / 失敗: 0件

### ドキュメント更新
- 更新ドキュメント一覧（なしの場合は「なし」）
```
